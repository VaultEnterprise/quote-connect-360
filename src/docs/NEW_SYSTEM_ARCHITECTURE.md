## Normalized Audit / Version Model
The rebuilt system should stop storing major audit/version behavior only inside JSON arrays on records.

Required normalized patterns:
- QuoteVersion entity for every recalculation or pricing-affecting edit.
- PlanRateVersion or AuditEntry coverage for rate changes.
- WorkflowEvent entity for stage/status transitions.
- QuoteTraceItem entity connecting quote outputs to census members, plans, rates, and contribution logic.
- EnrollmentElection lineage back to approved quote version and scenario plan.

## Workflow Architecture
### Case State Machine
Case is the operational anchor. Allowed transitions are enforced in a domain workflow service, not directly by page buttons.

### Census Readiness Workflow
States: uploaded → mapping → validating → validated / has_issues → archived.
Blocking rules feed quote readiness automatically.

### Quote State Machine
States: draft → in_progress → running → reviewed → approved → converted_to_enrollment / expired / error.
Every transition writes workflow and audit events.

### Proposal Workflow
States: draft → sent → viewed → approved / rejected / expired.
Proposal approval can trigger quote decision confirmation.

### Enrollment Workflow
States: scheduled → open → closing_soon → closed → finalized.
Input mapping is generated from approved quote decision service.

### Renewal Workflow
States: pre_renewal → marketed → options_prepared → employer_review → decision_made → install_renewal → active_renewal → completed.
Renewal decisions can trigger new quote generation or direct continuity workflows.

### Exception Workflow
States: new → triaged → in_progress → waiting_external → resolved / dismissed.
Auto-generated by policy services across domains.

## Recommended Rebuild File Organization
- `src/domain/<module>/...`
- `src/services/<module>/...`
- `src/components/<module>/...`
- `src/pages/<module>/...` or current page routes backed by module sections
- `src/lib/workflows/...`
- `src/lib/audit/...`
- `src/lib/integrations/...`
- `src/docs/...`

## Mandatory Rebuild Principles
- no page-owned business logic as final source of truth
- no duplicated pricing logic
- no implicit workflow transitions
- no hidden dependency assumptions
- no inline-only version history for critical financial workflows
- no integration UI without execution framework behind it
