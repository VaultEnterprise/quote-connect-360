## Section 1 — Final Gate 6D Scope

### Capability: Export Delivery History & Tracking

Gate 6D introduces a read-only, scoped, auditable view of all report export operations executed within the MGA platform. It enables MGA administrators and authorized managers to track, inspect, and understand the lifecycle of every export request generated by Gate 6C.

---

### In-Scope Features

| Feature | Description |
|---|---|
| View export history | Paginated list of all export requests within scoped MGA |
| Track export request status | Draft, processing, completed, failed, expired, cancelled |
| Track export file type | CSV, XLSX, PDF |
| Track export requester | User ID, email, role at time of request |
| Track export scope | MGA ID, Master Group ID, Case ID, Quote ID (where applicable) |
| Track export timestamps | requested_at, generated_at, downloaded_at, expires_at, cancelled_at |
| Track export success/failure state | Success + record count, or failure + reason code |
| Track expiration / artifact availability | Boolean: artifact_available, expires_at threshold check |
| Track cancellation / retry status | If cancellation or retry is later approved (deferred UI; logged if supported) |

---

### Explicitly Deferred (Out of Scope for Gate 6D)

| Deferred Capability | Reason |
|---|---|
| Report scheduling | Requires job scheduler infrastructure (Phase 2+) |
| Saved report templates | Requires new entity + permission model (Gate 6E+) |
| Advanced analytics dashboard expansion | Requires new KPI calculation engine |
| Automated email delivery | Requires notification/delivery infrastructure |
| External webhook delivery | Requires webhook registration + security signing |
| Bulk scheduled exports | Requires job queue + storage infrastructure |

## Section 3 — Proposed Data Model

### Recommended Approach

**Primary Source of Truth: ActivityLog entity (existing)**

Gate 6D uses the `ActivityLog` entity as the source of truth for export history records. This is consistent with Gate 6C's audit design and avoids data duplication.

A lightweight export history service will query, transform, and scope `ActivityLog` records tagged with export-related action types — without duplicating the underlying record payloads.

### Constraints

- ❌ Do NOT store exported content in history records
- ❌ Do NOT store signed URLs in history records (time-limited; cannot be safely persisted)
- ❌ Do NOT store private file URIs in history records
- ❌ Do NOT duplicate sensitive export payloads
- ✅ Store metadata only (who, what, when, outcome)

---

### Optional: ExportHistoryRecord (Lightweight Entity — Proposed)

If `ActivityLog` query patterns prove insufficient for the required filter/pagination needs, a lightweight ExportHistoryRecord entity may be introduced. **This entity is NOT created at planning stage.**

**Proposed fields (planning level only):**

| Field | Type | Description |
|---|---|---|
| `export_history_id` | string | Primary key |
| `export_request_id` | string | Correlation ID from Gate 6C export request |
| `mga_id` | string | MGA scope — required, enforced |
| `master_group_id` | string | MasterGroup scope — nullable |
| `case_id` | string | Case scope — nullable |
| `quote_id` | string | Quote scenario scope — nullable |
| `report_type` | string | case_summary, quote_scenario, census_member, audit_activity, mga_summary |
| `format` | string | csv, xlsx, pdf |
| `requested_by_user_id` | string | User email of requester |
| `requested_by_role` | string | Role at time of request |
| `status` | string | processing, completed, failed, expired, cancelled |
| `record_count` | number | Number of records in export |
| `requested_at` | datetime | Timestamp of request |
| `generated_at` | datetime | Timestamp of completed generation |
| `downloaded_at` | datetime | Timestamp of first download |
| `expires_at` | datetime | Artifact expiration threshold |
| `cancelled_at` | datetime | Timestamp of cancellation (if applicable) |
| `failure_reason_code` | string | Structured failure code (no PII) |
| `artifact_available` | boolean | Whether artifact can be accessed |

**Decision deferred:** Whether to create this entity or rely solely on ActivityLog will be determined at implementation planning time, after Gate 6C operator approval.

- ✅ All history reads MUST pass `scopeGate` validation

## Section 6 — Backend Contract

### Planned Backend Function: `mgaExportHistory`

All actions routed through a single validated entry point enforcing the same fail-closed authorization chain established in Gate 6C.

---

### Action: `listExportHistory`

**Purpose:** Return paginated export history records for a scoped MGA user.

| Property | Value |
|---|---|
| Required scope | Valid MGA ID, resolved via scopeGate |
| Required permission | `mga.reports.history.view` |
| Allowed roles | admin, platform_super_admin, mga_admin, mga_manager |
| Input validation | MGA ID required; date range optional; format/type optional filters |
| Failure states | 401 Unauthorized, 403 Forbidden (scope/permission failure), 400 Bad Request (invalid filter) |
| Audit event | `export_history_listed` |
| Safe return payload | Record list (metadata only; no artifact URLs; no file URIs) |
| Restricted fields excluded | artifact_url, file_uri, signed_url, private_storage_path, pii fields |

---

### Action: `getExportHistoryDetail`

**Purpose:** Return full detail of a single export history record.

| Property | Value |
|---|---|
| Required scope | Valid MGA ID + record must belong to requesting MGA |
| Required permission | `mga.reports.history.view` |
| Allowed roles | admin, platform_super_admin, mga_admin, mga_manager |
| Input validation | export_history_id required |
| Failure states | 401, 403, 404 (record not found in scope) |
| Audit event | `export_history_detail_viewed` |
| Safe return payload | Full record metadata; audit timeline; no URLs |
| Restricted fields excluded | All URL/URI fields; all PII |

---

### Action: `getExportAuditTrail`

**Purpose:** Return full audit trail for a specific export request.

| Property | Value |
|---|---|
| Required scope | Valid MGA ID + record must belong to requesting MGA |
| Required permission | `mga.reports.history.audit` |
| Allowed roles | admin, platform_super_admin, mga_admin |
| Input validation | export_request_id required |
| Failure states | 401, 403 (insufficient permission or scope), 404 |
| Audit event | `export_audit_trail_accessed` |
| Safe return payload | Audit event list (action, actor, timestamp, outcome); no raw payloads |
| Restricted fields excluded | Sensitive keywords redacted (per Gate 6C audit policy) |

---

### Action: `retryExport`

**Purpose:** Re-trigger generation of a failed export request.

| Property | Value |
|---|---|
| Required scope | Valid MGA ID + record must belong to requesting MGA |
| Required permission | `mga.reports.history.retry` |
| Allowed roles | admin, platform_super_admin, mga_admin |
| Input validation | export_history_id required; status must be `failed` |
| Failure states | 401, 403, 404, 409 Conflict (already processing or completed) |
| Audit event | `export_retry_requested` |
| Safe return payload | Updated record metadata |
| Restricted fields excluded | All URL/URI fields |
| **Gate:** ❌ DEFERRED — Not activated unless separately approved |

## Section 8 — Rollback Strategy

### Rollback Trigger

Set Gate 6D feature flag to `false` and redeploy.

```javascript
// File: components/mga/MGACaseWorkflowPanel.jsx (when implemented)
const MGA_EXPORT_HISTORY_ENABLED = false; // rollback switch
```

### Rollback Behavior

| System Component | Rollback Effect |
|---|---|
| Export History tab/panel | Hidden immediately |
| Backend history actions | Return `FEATURE_DISABLED` |
| ActivityLog records | Preserved — no deletions |
| ExportHistoryRecord entity (if created) | Records preserved; UI inaccessible |
| Gate 6C export generation | UNAFFECTED — independent feature flag |
| Gate 6B TXQuote Transmit | UNAFFECTED — zero interaction |
| Gate 6A Invite User | UNAFFECTED — zero interaction |

### Rollback Rules

- ✅ Disable Gate 6D feature flag → history UI hidden
- ✅ Backend history actions blocked
- ✅ Audit/history records preserved (not deleted)
- ❌ Do NOT delete audit or history records during rollback unless separately approved
- ❌ Do NOT affect Gate 6C export generation
- ❌ Do NOT affect Gate 6B TXQuote transmit
- ❌ Do NOT affect Gate 6A user invites

### Rollback Risk Level: 🟢 EXTREMELY LOW

- Flag-only change; no schema migrations required
- No data loss
- No breaking changes to existing gates
- Reversible in < 5 minutes
